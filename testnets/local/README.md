# Local Kestrel testnet

Generate at least four validator profiles with `cli validator-init NAME STAKE NETWORK_ADDRESS RPC_ADDRESS GOSSIP_ADDRESS OUTPUT_DIR` (this now also generates and persists that validator's libp2p gossip identity, in addition to its consensus key), combine their public `validator.json` documents into a JSON array, and use `cli genesis-create` to produce a local genesis file. `scripts/devnet-up.sh GENESIS_JSON` starts a passive genesis/RPC process for inspection.

To participate as a validator, start one process per genesis validator and supply its `--validator-id`, `--validator-key`, `--gossip-key`, and unique `--data-dir` in addition to its RPC address. The validator path runs the full production pipeline: real libp2p transaction gossip and `KestrelCast` shred relay, real consensus ordering, deferred execution, and an atomic durable commit — a transaction submitted to any validator's `kestrel_submitTransaction` RPC method is admitted, gossiped, ordered, executed, and committed identically on every validator. This is proven in automation across four separate OS processes (see `crates/node/tests/stage2_node_rpc_integration.rs`) and, for Byzantine-fault scenarios, across five (`crates/node/tests/stage_2_processes.rs`). It is still a local/loopback exercise, not a Stage 2 devnet: no socket-level latency/loss/relay death has been injected on the libp2p paths specifically, and no propagation/finality numbers have been measured across real machines or geography (see `docs/TECH_DEBT.md` TD-003).
